
# Adventure Works Sales Analysis

## Project Overview

Adventure Works Sales Analysis is an Excel-based business intelligence project built using four years of transactional sales data. The goal of this project was to transform raw data into an interactive and insightful dashboard that supports business decision-making.

This was a self-practice portfolio project created to strengthen practical skills in Excel, Power Query, Power Pivot, data modeling, and dashboard design.

The dataset was imported, cleaned, transformed, and modeled using Power Query, Power Pivot, and the Excel Data Model. Relationships were created between multiple tables to allow dynamic analysis across time, products, customers, geography, and demographics.

---

## Objective

The main objective of this project was to identify the key drivers of profit and analyze how business performance changes over time, across product categories, customer groups, and countries.

The analysis was designed to answer questions such as:
- Which products generate the highest profit?
- Which customers contribute the most to profitability?
- How does profit change over months, quarters, and years?
- Which countries perform best?
- Do age, gender, product color, or pricing type affect profitability?

## Data Modeling Approach

To handle the large dataset efficiently, the data was structured into six related tables:

- Fact Internet Sales
- Dim Sales Territory
- Dim Product
- Dim Geography
- Dim Date
- Dim Customer

The data was cleaned and transformed using Power Query before being loaded into the Excel Data Model. Relationships were created between the tables to support scalable and dynamic analysis.

---

## Workflow Followed

1. Imported raw transactional data using **Get Data**.
2. Cleaned and transformed the data using **Power Query**.
3. Split the dataset into separate fact and dimension tables.
4. Loaded all tables into the **Data Model**.
5. Created relationships between the tables.
6. Built calculated measures for analysis.
7. Used **M code** to calculate age.
8. Applied a macro to clear filters.
9. Linked both dashboards for seamless navigation.

## Dashboards Created

### Dashboard 1: Time Analysis

This dashboard focuses on business performance over time and includes:
- KPI comparison with previous year
- Yearly performance metrics
- Monthly profit trends
- Profit by week type
- Quarterly profit analysis
- Profit by weekday

This dashboard helps identify seasonality, performance cycles, and time-based patterns.

### Dashboard 2: Product and Customer Analysis

This dashboard focuses on profitability by product and customer segments and includes:
- Top 5 profitable products
- Top 5 profitable customers
- Profit by gender
- Profit by product color
- Profit by pricing type
- Country-wise profit map
- Profit by age group

This dashboard helps identify where profit is concentrated and which segments contribute most to business performance.

---

## Key Insights

- A small number of products and customers contribute a large share of total profit.
- Profit changes across months, quarters, and weekdays, showing clear time-based patterns.
- Some countries generate significantly more profit than others.
- Product attributes such as color and pricing type influence profitability.
- Customer age and gender provide useful segmentation insights.

---

## Business Recommendations

- Focus on high-performing products and customers to increase profitability.
- Use monthly and quarterly trends for better forecasting and planning.
- Strengthen marketing and sales efforts in high-profit countries.
- Review low-performing product categories, colors, and pricing types.
- Use customer segmentation to improve targeting strategies.
- Continue monitoring KPIs to track performance over time.
